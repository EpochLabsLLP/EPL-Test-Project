# template-sync

Compares the current project against the Epoch Labs project template and reports drift. Optionally applies updates.

## What It Does

1. Reads `TEMPLATE_MANIFEST.json` from the template repo
2. Compares every declared file against the current project
3. Reports what's missing, drifted, up-to-date, or project-owned
4. Optionally applies updates (overwrite infrastructure/template files, create missing files)

## File Ownership Categories

| Category | Sync Behavior | Examples |
|----------|--------------|---------|
| **infrastructure** | Always overwritten | Hooks, rules, skills, scripts |
| **template** | Always overwritten | TEMPLATE_*.md files |
| **scaffolding** | Created once, never overwritten | CLAUDE.md, settings.json |
| **generated** | Skipped (regenerated by tools) | Work_Ledger.md |

## When to Use

- After the template repo has been updated with new hooks, rules, or skills
- When onboarding a new project that may be missing template components
- To audit an existing project for template drift
- When Nathan asks to "sync the template" or "update from template"

## How to Invoke

```
/template-sync              # Dry-run report (no changes)
/template-sync --apply      # Apply changes (with backup)
```

## Execution

### Step 1: Locate the Template Repo

The template repo is at a known path. Search for `TEMPLATE_MANIFEST.json` in:
1. `$CLAUDE_PROJECT_DIR/../_ProjectTemplate/` (sibling directory)
2. `C:/Claude Folder/Epoch_Labs/Epoch Labs Project Template/_ProjectTemplate/`

If not found, ask Nathan for the template repo path.

### Step 2: Run the Sync Engine

```bash
python "$TEMPLATE_DIR/template_sync.py" "$TEMPLATE_DIR" "$CLAUDE_PROJECT_DIR"
```

Or with `--apply`:

```bash
python "$TEMPLATE_DIR/template_sync.py" "$TEMPLATE_DIR" "$CLAUDE_PROJECT_DIR" --apply
```

### Step 3: Present Results

Show the sync report to Nathan. Key sections:
- **WILL CREATE** — Missing files that will be added
- **WILL UPDATE** — Drifted infrastructure/template files that will be overwritten (backup first)
- **SKIPPED** — Project-owned files (CLAUDE.md, settings.json) that are never touched
- **UP TO DATE** — Files that match the template

### Step 4: Confirm Before Apply

If `--apply` was NOT passed and actions are needed, ask:
"The template has updates. Run `/template-sync --apply` to apply them?"

## Safety

- **Never deletes files** — only creates or overwrites
- **Never modifies scaffolding** — CLAUDE.md, settings.json are always skipped
- **Always backs up** — overwritten files saved to `.template_backup/{timestamp}/`
- **Dry-run by default** — must explicitly pass `--apply` to make changes
